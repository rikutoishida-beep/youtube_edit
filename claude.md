@input_srt　@output_srt

あなたはプロの「動画テロップディレクター」です。
提供されたSRTデータから話者数を自動判別し、視聴者が最も読みやすい形式へ最適化した上で、話者ごとに個別のファイルを出力してください。

以下の【2つのステップ】を厳守して実行してください。

【ステップ1：修正・削除方針の確認】
SRTデータを読み込み、修正作業に入る前に、以下の調査結果をリストアップして提示してください。
1. 判別した話者リスト（データ内に存在するすべての話者を特定）
2. 抽出した固有名詞・専門用語（人名、サービス名、ゲーム名、専門資格など）
3. 【重要】ケバ取り・削除候補のリスト：
　テキスト内で多用されている「えー」「あのー」などのフィラーや、言い淀み、削除しても文脈に影響がないと思われる言葉をサンプルとして抽出してください。
4. 表記ゆれの修正案

※上記リストを提示し、「この方針でケバ取りと固有名詞の修正を行い、最終出力を開始してよろしいでしょうか？」とユーザーに確認してください。

【ステップ2：一括最適化処理と話者別出力】
ユーザーの承認（または個別指示）を得たら、以下の【処理ルール】を適用し、話者ごとに個別のSRTデータを出力してください。

■ ルール1：テキストの校正・ケバ取り
・誤変換を文脈に沿って正確に修正してください。
・【固有名詞の聖域化】: ステップ1で確定した固有名詞やカタカナ語は一字一句変えずに保持してください。
・【ケバ取りの実行】: ステップ1で承認された「えー」「あのー」等のフィラーのみ削除してください。
・【ニュアンスの保護】: 「うん」「はい」「なるほど」などの相槌、および「〜みたいな」「〜かな」「やっぱり」等の口語表現は、指示がない限り絶対に削除しないでください。
・【数字の表記】: 慣用句（一言、二度見、四苦八苦など）以外の数字は半角数字（0-9）で統一してください。

■ ルール2：1文の原則と20文字分割
・1つのブロックには「基本的に一文のみ」を含めてください。
・【20文字の壁】: 1文が15文字を超える場合は、必ず読点（、）の位置で分割してください。
・読点がない場合も、15文字を超えるなら自然な文節で分割してください。
・不自然に分割している文章はテキストを移動させることで修正してください。
・修正後に不自然でないか再チェックしてください。

■ ルール3：出力形式（話者データの完全除去）
・話者（話者 1, 2...）ごとに個別のSRTデータを生成してください。
・出力するSRTのテキスト部分からは、「話者1：」や「[話者2]」といった話者特定タグをすべて削除し、純粋なセリフのみにしてください。
・各ファイル内の連番（Index）は「1」から振り直してください。

■ ルール4：タイムコード再計算とバリデーション
・分割時は、文字数比率に合わせてタイムコードをミリ秒単位で按分してください。
・【1秒セーフティ】: 分割後の1ブロックの表示時間が「1秒未満」になる場合は、無理に分割せず前後のブロックと結合してください。
・出力前に【見本（00:00:05,140 --> 00:00:08,260）】と書式が一致するか自己検証してください。

【出力形式】
「話者1（〇〇）用SRT」「話者2（〇〇）用SRT」のようにタイトルを明記し、srtファイルを分けて出力してください。

## 実行環境と使用方法

### 環境要件
- **Python**: 3.8以上
- **OS**: Linux/macOS/Windows
- **外部ライブラリ**: 不要（標準ライブラリのみ使用）

### 基本的な実行方法

```bash
# ディレクトリ構造の準備
mkdir -p input_srt output_srt

# 処理スクリプトの実行
python3 process_srt.py
```

### 事前準備
1. 校正したいSRTファイルを `input_srt/` フォルダに配置
2. SRTファイルには「話者 1：」「話者 2：」形式で話者タグが含まれていることを確認
3. `output_srt/` フォルダが存在しない場合は自動生成されます

## 入出力仕様

### 入力ファイル形式
- **ファイル形式**: SRT（SubRip形式）
- **エンコーディング**: UTF-8
- **話者表記**: 各ブロックの最初に「話者 1：」「話者 2：」などのタグが存在
- **タイムコード形式**: HH:MM:SS,mmm --> HH:MM:SS,mmm

**入力例**:
```
1
00:00:05,140 --> 00:00:07,200
話者 1：こんにちは。

2
00:00:07,500 --> 00:00:10,300
話者 2：こんにちは。お疲れ様です。
```

### 出力ファイル形式
- **出力先**: `output_srt/` フォルダ
- **ファイル名形式**: `話者X（名前）.srt`（例：話者1（ごますけ）.srt）
- **エンコーディング**: UTF-8
- **特徴**:
  - 話者タグはなし（純粋なセリフのみ）
  - テキスト長は全て20文字以下
  - 連番は「1」から開始
  - 各ブロックの表示時間は最短1秒を保証

**出力例**:
```
1
00:00:05,140 --> 00:00:06,240
こんにちは。

2
00:00:06,240 --> 00:00:10,300
お疲れ様です。
```

## カスタマイズガイド

### 名前の修正ルールの拡張

`process_srt.py` の `apply_corrections()` 関数内で修正辞書を編集してください：

```python
corrections = {
    'シリン': '白臨',      # 「シリンさん」→「白臨さん」
    'しりん': '白臨',      # 「しりんさん」→「白臨さん」
    'ゴマスケ': 'ごますけ', # 「ゴマスケさん」→「ごますけさん」
    'こた': 'こと',        # 「こたさん」→「ことさん」
    # 新しい修正ルールをここに追加
    '新規誤字': '正しい字',
}
```

### フィラー・除去対象の変更

`remove_fillers()` 関数を編集：

```python
filler_patterns = [
    (r'あの、', ''),      # 「あの、」を削除
    (r'えー', ''),         # 「えー」を削除
    # 新しいフィラーパターンを追加
    (r'パターン', '置換後'),
]
```

### テキスト分割の文字数調整

`split_text_for_display()` 関数の以下の値を変更：

```python
max_chars = 15      # ターゲット文字数（通常は15）
hard_max = 20       # 絶対上限（通常は20）
```

### 話者の自動判別ロジック

現在のロジックは「話者 1：」「話者 2：」形式を想定しています。
異なるフォーマット（例：「[話者A]」）の場合は `parse_srt()` 関数のパターンマッチングを修正してください：

```python
# 現在のパターン
speaker_pattern = r'話者\s*([0-9]+)[:：]'

# 例：[話者A] 形式の場合
speaker_pattern = r'\[話者([A-Z])\]'
```

## よくある問題と対処法

| 問題 | 原因 | 対処法 |
|------|------|--------|
| **タイムコードのズレが大きい** | 元のSRTが断片化され、分割時に時間配分が不正確 | 元のSRTファイルのタイムコードが正確であることを確認。大幅な分割が必要な場合は、手動でタイムコードを微調整 |
| **固有名詞が誤字のままになっている** | `apply_corrections()` の辞書に登録されていない | 該当する誤字と正字の組み合わせを `corrections` 辞書に追加 |
| **フィラーが削除されない** | フィラーが前後の文脈と結合している | `remove_fillers()` に新しいパターンを追加（例：「あの、」だけでなく「、あの」も対象に） |
| **テキストが20文字を超えている** | 分割ロジックが読点を認識していない | 分割対象の読点位置を確認し、手動で読点を追加または UNSPLITTABLE 辞書を調整 |
| **相槌「うん」が削除されている** | フィラー削除時に相槌を誤検出 | `remove_fillers()` のパターンを確認し、相槌を除外する条件を追加 |
| **世の中」などが「世/ の/中」に分割される** | UNSPLITTABLE 辞書に登録されていない | 分割してはいけない複合語を UNSPLITTABLE 辞書に追加 |
| **出力ファイルが生成されない** | `input_srt/` フォルダが空または読み取り権限がない | ファイルの存在と権限を確認。パスが正しいことを確認 |
| **話者が3人以上の場合** | 現在のスクリプトは2話者を想定 | `parse_srt()` の話者数判定ロジックを拡張し、出力形式を調整 |

## 品質チェックリスト

処理完了後、以下の項目をチェックしてください：

### ✓ 出力ファイルの確認
- [ ] `output_srt/` フォルダに期待する話者別ファイルが存在する
- [ ] ファイル名が「話者X（名前）.srt」形式である
- [ ] ファイルが UTF-8 エンコーディングである

### ✓ テキスト内容の確認
- [ ] 全ブロックのテキストが20文字以下である
- [ ] 話者タグ（「話者 1：」など）が完全に削除されている
- [ ] 固有名詞が正しく修正されている
- [ ] 指定されたフィラーのみ削除されている
- [ ] 相槌（「うん」「はい」）が保持されている

### ✓ タイムコードの確認
- [ ] 全ブロックのタイムコード形式が「HH:MM:SS,mmm」である
- [ ] 開始時刻 < 終了時刻 の関係が全ブロックで成立している
- [ ] 各ブロックの表示時間が最短1秒である
- [ ] 連続するブロック間にタイムコードの重複やギャップがない

### ✓ 連番とブロック数の確認
- [ ] 各ファイルの連番が「1」から開始している
- [ ] 連番が1つ飛ばしなく続いている
- [ ] 話者1と話者2の出力ブロック数の合計が入力の処理済みブロック数に近い

### ✓ 自然性の確認
- [ ] 分割したテキストが不自然な箇所がない
- [ ] 意味が通じなくなっている箇所がない
- [ ] 句点「。」の位置が適切である

### 自動検証項目

スクリプトは以下を自動的に検証します：

1. **ブロック数の確認**: 入力と出力でブロック数が妥当か
2. **テキスト長の確認**: 全ブロックが20文字以下か
3. **タイムコード形式**: 正規表現で形式チェック
4. **連番の連続性**: 各ファイルで1から順に採番されているか
5. **話者タグの除去**: 出力テキストに「話者」という文字が存在しないか

### 出力ファイルの手動検証方法

任意のテキストエディタでファイルを開き、以下をチェック：

```bash
# テキスト長の確認（各行が20文字以下）
wc -L output_srt/話者1*.srt

# 話者タグが残っていないか確認
grep -n "話者\|Speaker" output_srt/*.srt

# タイムコード形式の確認
grep -E "^[0-9]{2}:[0-9]{2}:[0-9]{2},[0-9]{3}" output_srt/*.srt | head -5
```
